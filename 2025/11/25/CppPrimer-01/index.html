变量和基本类型 简介

数据类型是程序的基础：它告诉我们数据的意义以及我们能在数据上执行的操作。

​ C++语言支持广泛的语言类型。它定义了几种基本内置类型，同时也为程序员提供了自定义数据类型的机制。基于此，**C++标准库定义了一些更加复杂的数据类型**。本章将主要讲述内置类型，并带领大家初步了解C++语言是如何支持更复杂数据类型的。

基本内置类型

算术类型

整型和浮点型

​ 算术类型分为两类：整型和浮点型。其尺寸在不同机器上有所差别，下表是C++标准规定的尺寸最小值：

类型 含义 最小尺寸
bool 布尔类型 未定义
char 字符 8位
wchar_t 宽字符 16位
char16_t Unicode字符 16位
char32_t Unicode字符 32位
short 短整型 16位
int 整型 16位
long 长整型 32位
long long 长整型 64位
float 单精度浮点数 6位有效数字
double 双精度浮点数 10位有效数字
long double 扩展精度浮点数 10位有效数字

​ 可总结以下几点：

  1. 布尔类型的取值为真（true）或假（false）。
  2. 基本字符类型为char，其空间确保能存放机器基本字符集中任意字符对应的数字值；其它字符类型用于扩展字符集。
  3. 整型表示不同尺寸的整数。
  4. 浮点型可表示单精度、双精度和扩展精度值。

带符号类型和无符号类型

​ 除去布尔型和扩展字符型，其它整型可以分为带符号类型与不带符号类型。带符号类型存在正、负与零，而无符号类型仅能表示正与零。

​ 对于int/short/long/long long均为带符号类型，可以在前面加上unsigned得到无符号类型。

​ 而字符型分为了char/signed char/unsigned char。需要注意的是char与signed char并不一样！

选类型建议：

  1. 明确知晓数值不可能为负的时候，采用无符号类型；
  2. 一般使用int进行整数运算，如果数值范围超过了int数值范围，请采用long long；
  3. 算术表达式中不要使用bool和char（char的符号类型是不定的）；
  4. 执行浮点数运算选用double;

类型转换

​ 对象的类型定义了对象能包含的数据和能参与的运算，其中转换运算被大多数类型支持。类型所能表示的值范围决定了转换的过程：

  1. 非布尔类型赋给布尔类型时，初始值为0则为false否则为true；
  2. 布尔类型赋给非布尔类型时，false为0，true为1;
  3. 浮点数赋给整数类型时，保留小数点前的部分；
  4. 整数值赋给浮点类型时，小数部分记作0；
  5. 赋给无符号类型超出表示范围的值时，将会得到对表示数值总数取模后的结果（-1赋值给unsigned char，则其值为255）；
  6. 如果给带符号类型赋值超出表示范围的值，则结果未定义！

切勿混用带符号类型和无符号类型：

​ 表达式中既有带符号类型，又有无符号类型，则带符号类型会自动转为无符号类型，如果带符号类型为负则会出现异常结果！

字面值常量

​ 字面值常量（literal）是指在程序代码中直接写出来、不会再被计算或解析、其值在编译/运行时保持不变的常量。

  1. 整型字面值：存在十进制、八进制、十六进制三种形式，其中以0开头的为八进制，以0x或0X开头的为十六进制；
  2. 浮点型字面值：一个小数或者以科学计数法表示的指数；
  3. 字符字面值：单引号括起来的单个字符（'a'）;
  4. 字符串字面值：双引号括起来的零个字符或者多个字符；
  5. 布尔字面值：true/false;
  6. 指针字面值：nullptr;

转义序列

转义序列 含义
\n 换行
\r 回车
\t 水平制表符
\v 垂直制表符
\b 退格
\f 换页
\a 响铃
\\ 反斜杠
\' 单引号
\" 双引号

​ 上表是一些常用的转义序列，我们也可以用泛化的转义序列：

  1. 八进制转义序列：\ooo,o是八进制数字，\后可接一到三位，超出则判定为普通字符；
  2. 十六进制转义序列：\xhh...，h是一个或者多个十六进制数字，不限长度；

指定字面值类型

​ 可以通过前缀与后缀，改变字面值默认类型：

  1. 字符和字符串字面值：

    前缀 含义 类型
    u Unicode 16字符 char16_t
    U Unicode 32字符 char32_t
    L 宽字符 wchar_t
    u8 UTF-8 char

  2. 整型字面值：

    后缀 最小匹配类型
    u/U unsigned
    l/L long
    ll/LL long long

  3. 浮点型字面值：

    后缀 类型
    f/F float
    l/L long double

变量

​ 变量提供一个具名的、可供程序操作的存储空间。对C++程序员来说，对象与变量可以互换使用。

通常情况下，对象是指一块能存储并具有某种类型的内存空间。

变量定义

​ 变量定义的基本形式是：首先是类型说明符，随后紧跟一个或多个变量名组成的列表（变量名用逗号分隔），最后以分号结束。定义时还可以同时赋予初值。

初始值

​ 当对象在创建时获得了一个特定的值，我们说这个对象被初始化了。

初始化不是赋值！初始化的含义是创建变量时赋予一个初始值，而赋值的含义是把对象当前值擦除后，以新值替代。

列表初始化

​ C++中初始化方法多样：

​ C++11标准使得用花括号来初始化变量得到了全面应用，这种初始化方式称为列表初始化。注意，如果我们使用了列表初始化且存在丢失信息的风险时，编译器将报错：

默认初始化

​ 如果指定变量时没有指定初值，则变量被默认初始化。默认值不仅由变量类型决定，也由定义变量的位置决定。

定义于函数体内的内置类型的对象如果没有初始化，则其值未定义。类的对象如果没有显式地初始化，则其值由类定义。

变量声明与定义的关系

​ 为了允许把程序拆分多个逻辑部分编写，C++支持分离式编译机制。

​ 为了支持分离式编译机制，C++语言将声明和定义区分开来。声明使得名字为程序所知，一个文件如果想使用别处定义的名字，则必须包含对名字的声明；而定义负责创建与名字关联的实体。变量声明规定了变量的类型和名字，而定义在这之上还申请了存储空间（也可能赋予了初始值）。

​ 如果想声明一个变量而非定义它，就在变量名前添加关键词extern，而且不要显式初始化该变量：

​ 任何包含了显式初始化的声明即成为了定义！而在函数体内部，如果试图初始化一个由extern关键字编辑的变量，将引发错误。变量只可以被定义一次，但可以被声明多次。

标识符

​ C++标识符由字母、数字和下划线组成，其中必须以字母或者下划线开头。标识符的长度没有限制，但是对大小写字母敏感。同时C++保留了一些名字供语言本身使用，这些名字不能被用作标识符。

​ 用户自定义的标识符中，不能连续出现两个下划线，也不能以下划线紧连大写字母开头；定义在函数体外的标识符也不能以下划线开头。

变量命名规范

  1. 标识符要能够体现实际含义；
  2. 变量名一般用小写字母，如index；
  3. 用户自定义类名一般以大写字母开头；
  4. 多个单词组成的标识符，需要在单词间具有明显区分；

名字的作用域

​ 作用域是程序的一部分，在其中名字有其特定含义。C++中大多数作用域以花括号分隔。名字的有效区域始于名字的声明语句，以声明语句所在的作用域末端结束。

​ 名字main在所有花括号之外，因此其具有全局作用域；一旦声明，全局作用域内的名字在整个程序的范围内都可使用。名字sum定义于main函数限定的作用域之内，因此声明开始至main结束均可访问，离开相应的代码块后就无法访问了，因此其具有块作用域。

嵌套的作用域

​ 作用域能彼此包含，被包含的叫做内层作用域，包含其它作用域的叫做外层作用域。

​ 内层作用域能重新定义外层作用域的已有名字，且可以访问外层作用域的已有变量。

复合类型

​ 复合类型是指基于其它类型定义的类型。

引用

​ 引用为对象起了另外一个名字，通过将声明符写成&d的形式来定义引用类型，其中d是声明的变量名：

​ 在定义引用时，程序把引用和它的初始值绑定到一起，而不是将初始值拷贝到引用，一旦初始化完成，引用将和它的初始值对象一直绑定在一起，因为无法令引用重新绑定到另外一个对象，因此引用必须初始化。

引用并非对象，相反的，它只是为一个已经存在的对象所起的另外一个名字。

​ 获取引用的值，实际上是获取了与引用绑定的对象的值，因为引用并非对象，因此不能定义引用的引用。

引用的定义

​ 允许在一条语句中定义多个引用，其中每个引用标识符都必须以符号&开头。

​ 一般情况下，引用的类型都要和与之绑定的对象严格匹配，并且，引用只能绑定到对象上，而不能与字面值或某个表达式的计算结果绑定在一起。

指针

​ 指针是指向另外一种类型的复合类型。指针与引用类似，同样实现了对其他对象的间接访问，然而指针与引用又有很多不同点。

指针与引用的不同：

  1. 指针本身是对象，而引用并非对象；
  2. 指针无须在定义时赋初值；

获取对象的地址

​ 指针存放某个对象的地址，获取对象的地址需要使用取地址符&。一般情况下，指针的类型需要与其所指对象严格匹配。

指针值

​ 指针的值应属下列4种状态之一：

  1. 指向一个对象；
  2. 指向紧邻对象所占空间的下一个位置；
  3. 空指针；
  4. 无效指针；

利用指针访问对象

​ 如果指针指向一个对象，则允许使用解引用符*来访问该对象。

空指针

​ 空指针不指向任何对象。得到空指针最直接的方法是用字面值nullptr初始化指针。

nullptr和NULL的不同：

  1. NULL是C定义的预处理变量，而nullptr是一个关键字；
  2. NULL没有明确的类型，nullptr类型是std::nullptr_t，独立类型；

void*指针

​ void*是一种特殊的指针类型，可用于存放任意对象的地址。
