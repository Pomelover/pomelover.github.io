第一章节：让自己习惯C++

条款01：视C++为一个语言联邦

View C++ as a federation of languages.

​ 条款01是对C++的一个总结性介绍，作者建议将C++视为一个多重范型编程语言（同时支持过程形式、面向对象形式、函数形式、泛型形式、元编程形式的语言）。在每个次语言内，守则是通用的，但是在不同的次语言内守则可能会发生变化。

​ C++主要有四种次语言：

  • C：C++仍然是以C为语言基础；
  • Object-Oriented C++：C++的最初名称即为 C with Classes，为了实现面向对象，C++引入了classes、封装、继承、多态等实施策略；
  • Template C++：Template C++是C++泛型编程部分，它带来了所谓的模板元编程（template metaprogramming, TMP）；
  • STL：STL全称为 Standard Template Library，它提供了泛型算法与数据结构的模板化实现；

总结

C++高效编程守则视状况而变化，取决于你采用了C++的哪一部分。

条款03：尽可能使用const

Use const whenever possible.

​ const允许程序员实现一种语义上的约束：特定的对象不应该被修改。

顶层指针与底层指针

​ const最为困惑的语法是顶层指针与底层指针。

  • 顶层指针（指针常量）：指针本身是常量;
  • 底层指针（常量指针）：指针指向的对象是常量；

​ 我一般是按照以下规则记的：就近原则，const离谁近修饰谁。

const成员函数

​ const修饰成员函数目的是为了标明这个成员函数可能会被const对象调用。const成员函数在类中十分重要，原因如下：

  1. 使类的接口更容易被理解。利用const修饰符可以区分函数是否能改变对象；
  2. 与const对象配合工作。const成员函数可以在 常量对象上调用。通过 const 修饰的成员函数，确保了调用该函数时不会改变对象的状态。

​ 同时，const作为修饰符可以重载成员函数：

Bitwise Constness V.S. Logical Constness

​ 而一个成员函数被const修饰时，究竟意味着什么呢？作者说这是一个哲学问题（可能类似唯物主义和唯心主义=v=）：

  • Bitwise Constness：对象本身的所有数据成员在方法中都不能被修改。
  • Logical Constness：对象的逻辑状态保持不变，但对象的内部数据成员 可以修改。

​ 注意：编译器采取Bitwise Constness，如果你想实现Logical Constness你需要使用mutable关键词进行修饰。

总结

  1. 将某些变量声明为const可帮助编译器侦测出错误用法。const可被施加于任何作用域内的对象、函数参数、函数返回类型、成员函数本体；
  2. 编译器强制实施Bitwise Constness，但在编写程序时应使用Logical Constness；
  3. 当const和non-const成员函数有着实质等价的实现时，令non-const版本调用const版本可避免代码重复（反向是不安全的）。
